What is Version Control?

Version control is a system that records changes to files over time, allowing you to recall specific versions later.

Why use it?

  • Track changes and history.
  • Collaborate with others.
  • Recover from mistakes.
  • Manage multiple versions of a project.

Types of Version Control Systems:

  • Centralized: SVN, CVS
  • Distributed: Git, Mercurial

Subversion (SVN)

A centralized version control system.

Why use SVN?

  • Centralized repository for easy management.
  • Strong support for binary files.
  • Fine-grained access control.
  • Atomic commits ensure consistency.
        
# Install SVN
sudo apt-get install subversion

# Create a Repository
svnadmin create /path/to/myrepo

# Checkout a Working Copy
svn checkout file:///path/to/myrepo myworkingcopy

SVN Workflow

Basic workflow for using SVN.

        
# Add and Commit Files
svn add file.txt
svn commit -m "Added file.txt"

# Update Working Copy
svn update

# Resolve Conflicts
svn resolve --accept working file.txt

# View Logs
svn log

# Revert Changes
svn revert file.txt

Branching and Merging:

        
# Create a Branch
svn copy http://svn.example.com/repo/trunk http://svn.example.com/repo/branches/my-branch -m "Creating a branch"

# Switch to Branch
svn switch http://svn.example.com/repo/branches/my-branch

# Merge Changes
svn merge http://svn.example.com/repo/trunk

Mercurial (Hg)

A distributed version control system.

Why use Mercurial?

  • Every user has a full copy of the repository.
  • Works offline.
  • Lightweight and fast.
  • Simple and intuitive commands.
        
# Install Mercurial
sudo apt-get install mercurial

# Initialize a Repository
hg init myproject

# Clone a Remote Repository
hg clone https://hg.example.com/remote-repo local-copy

Mercurial Workflow

Basic workflow for using Mercurial.

        
# Add and Commit Files
hg add file.txt
hg commit -m "Added file.txt"

# Push Changes
hg push

# Pull Changes
hg pull

# View Logs
hg log

# Revert Changes
hg revert file.txt

Branching and Merging:

        
# Create a Branch
hg branch my-branch

# Switch to Branch
hg update my-branch

# Merge Changes
hg merge

SVN vs. Mercurial

Key differences between the two systems.

        
- SVN: Centralized, requires a server.
- Mercurial: Distributed, works offline.
- SVN: Directory-based branching.
- Mercurial: Lightweight branching.
- SVN: Better for binary files.
- Mercurial: Faster and more scalable.
- SVN: Requires network access for most operations.
- Mercurial: Most operations are local.

When to Use Which?

Use SVN if:

  • You need centralized control.
  • Your project has many binary files.
  • You need fine-grained access control.
  • You prefer a simpler, server-based workflow.

Use Mercurial if:

  • You need a distributed system.
  • You want to work offline.
  • You need fast and lightweight branching.
  • You prefer a more modern and flexible workflow.

Conclusion

Version control is essential for modern software development. Choose SVN for centralized workflows and Mercurial for distributed workflows.
